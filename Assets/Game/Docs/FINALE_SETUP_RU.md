# Настройка финала и столов подношений

Этот документ описывает, как через Inspector настроить Room 4 и Room 5 без правок C#.

## Room 4: стол открывает проход

Главный стол комнаты 4 находится внутри `Level-4-v3`. На нем стоит `OfferingTableInteractable`. После размещения всех нужных предметов компонент вызывает событие `On Completed`.

`On Completed` - это UnityEvent, то есть список действий, которые Unity выполнит после заполнения стола. Само событие ничего не делает, пока в нем не назначены объект и функция.

Если в строке события видно `No Function`, это означает одно из двух:

- строка события пустая;
- объект уже перетащили в поле, но функцию еще не выбрали.

Для открытия прохода должно быть так:

1. В `On Completed` нажать `+`.
2. Перетащить объект с компонентом `Room04PassageOpener` в поле объекта.
3. В выпадающем списке выбрать `Room04PassageOpener -> Play`.

## Room04PassageOpener

`Room04PassageOpener` больше не ищет секции автоматически. Все движущиеся части стола назначаются вручную в `Panel Motions`.

Поля:

- `Panel Motions` - список секций, которые будут двигаться.
- `Target` - transform конкретной секции стола.
- `Position Space` - в каких осях применять смещение. Обычно ставь `World`, чтобы `Y` всегда был мировым вверх/вниз. `Local` нужен только если специально хочешь двигать секцию по ее локальным осям.
- `Position Offset` - смещение от стартовой позиции. При `Position Space = World` значение `{ x: 0, y: -0.05, z: 0 }` опустит секцию вниз по миру на 5 сантиметров.
- `Euler Offset` - дополнительный поворот в градусах. Для чистого движения вверх/вниз оставь `{ x: 0, y: 0, z: 0 }`.
- `Delay` - задержка только для этой секции.
- `Start Delay` - задержка перед всей анимацией.
- `Panel Move Duration` - длительность движения.
- `Panel Motion Curve` - кривая плавности.
- `Play Only Once` - не запускать анимацию повторно.
- `Log Opening` - писать в Console, сколько назначенных секций было найдено при запуске.

В Room 4 сейчас назначены четыре реальные секции стола, которые ты подключил вручную. Для них выставлено:

- `Position Space = World`;
- `Position Offset = { x: 0, y: -0.05, z: 0 }`;
- `Euler Offset = { x: 0, y: 0, z: 0 }`.

Это значит: секции должны только слегка опуститься вниз по мировой оси Y и не должны крутиться. Если хочешь движение вверх, поставь `Position Offset.y` положительным, например `0.05`. Если хочешь более заметный тест, временно поставь `-0.2` или `-0.3`.

В Console после заполнения стола должна появиться строка:

`Room04PassageOpener: captured 4 assigned panel motions`

Если там `0 assigned panel motions`, значит в `Panel Motions` не назначены `Target`.

## Почему секции могли улетать или крутиться

У импортированных FBX-частей локальные оси часто повернуты. Поэтому смещение по локальному `Y` может визуально выглядеть как движение в сторону или за сцену. Для таких секций используй `Position Space = World`.

Поворот задается только полем `Euler Offset`. Если секции крутятся, проверь, что у каждой строки `Euler Offset` стоит `{ x: 0, y: 0, z: 0 }`.

Старый вспомогательный компонент `InspectorObjectSequence` на префабе `PF_Room04PassageOpener` больше не нужен для этого прохода. Основная настройка теперь только в `Room04PassageOpener -> Panel Motions`.

## Как настроить свои секции

1. Выбери объект `Room04PassageOpener`.
2. В `Panel Motions` поставь нужный размер массива.
3. Перетащи каждую секцию стола в `Target`.
4. Поставь `Position Space = World`.
5. Задай `Position Offset`.
6. Оставь `Euler Offset = 0,0,0`, если нужен только сдвиг без вращения.
7. Запусти Play Mode и заполни offering table.

## Room 5: финальные столы

1. В комнате 5 должны быть четыре `OfferingTableInteractable`.
2. На каждом столе должен быть один required item: `spirit`, `mind`, `body`, `soul`.
3. В `OfferingTableCompletionGroup.Tables` назначь все четыре финальные стола.
4. В `On All Completed` должен быть вызов `FinaleSequenceController -> Play`.

## Smoke Test

1. Открой `Assets/Scenes/Room_v3.unity`.
2. Запусти Play Mode.
3. В Room 4 положи три нужных элемента на offering table.
4. Проверь, что в Console есть `captured 4 assigned panel motions`.
5. Проверь, что четыре секции стола двигаются только вверх/вниз и не вращаются.
6. Проверь, что `CubeHole` исчезает, если включен `Hide Cube Hole When Opened`.
